# Night Audit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Night Audit — a sequential, transactional, irreversible end-of-day closure process that posts daily room charges, processes no-shows, and advances the property's business date.

**Architecture:** Night Audit is a multi-step queued job (`NightAuditJob`) dispatched from the API. Steps are atomic: if any step fails, the audit fails and must be re-run. The `properties.business_date` only advances on success. The `night_audit_logs` table is immutable (no updates after insert). The pre-audit checklist is served via a read-only API endpoint before the audit is triggered.

**Tech Stack:** Laravel 13, Laravel Horizon (queued job), MySQL 8 transactions, Pest PHP.

**Depends on:** Phase 3 (Reservations), Phase 5 (Cashiering / Folio — PostRoomChargeAction), Phase 6 (Housekeeping — auto-dirty on checkout).

```sql
night_audit_logs  -- Immutable record per business_date (no updates after creation)
```

---

## Audit Steps (enforced sequence)

1. Pre-audit checklist (read-only, surfaces unresolved items)
2. Post daily room charges to all `checked_in` reservations
3. Process no-shows (`confirmed` reservations not yet checked in on `business_date`)
4. Advance `properties.business_date + 1`
5. Write immutable `night_audit_log` record
