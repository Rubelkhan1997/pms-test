# POS (Point of Sale) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the POS module for outlet sales — create orders, add items, and post charges directly to a checked-in guest's folio. Supports both room-linked and direct (walk-in) sales.

**Architecture:** POS orders reference a `reservation_id` (nullable for walk-ins). "Post to Folio" creates a `folio_item` with `reference_type = pos_order` and `reference_id = pos_order.id`, then sets `pos_order.status = posted`. The Phase 5 Cashiering folio engine handles the actual financial accounting.

**Tech Stack:** Laravel 13, Pest PHP.

**Depends on:** Phase 5 (Cashiering — folio_items table and PostChargeAction must exist).
