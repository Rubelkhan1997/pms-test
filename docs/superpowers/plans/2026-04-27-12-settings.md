# Settings (Tenant-Level) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build all tenant-level settings screens — Users & Roles, Room Types, Amenities, Taxes, Payment Methods, Email Templates, Policies, Travel Agents, Business Sources, and Company Database.

**Architecture:** All data is scoped to the tenant DB. Roles and permissions use `spatie/laravel-permission` (already installed). Tax codes are referenced by folio item type during posting. Payment methods drive the Post Payment form dropdown. Email templates are used by notification events.

**Tech Stack:** Laravel 13, Spatie Permission, Pest PHP.

**Depends on:** Phase 2 (Onboarding — room_types, cancellation_policies, rate_plans exist and are editable here).

---

## Key Tables to Create

```sql
amenities             -- Master list for property
property_amenities    -- Pivot: property ↔ amenities
tax_codes             -- VAT, Service Charge, etc.
payment_methods       -- Cash, Visa, Bank Transfer, etc.
email_templates       -- Transactional email bodies per event
travel_agents         -- Agent profiles + commission
business_sources      -- Market segment tracking
companies             -- Corporate client profiles for direct billing
```
