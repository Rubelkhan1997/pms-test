## Module Info
- Module Name:      [MODULE_NAME]      ← Namespace/Folder  → e.g. FrontDesk
- Feature Name:     [FEATURE_NAME]     ← Sub-feature name  → e.g. Reservation
- Model Name:       [MODEL_NAME]       ← Model class name  → e.g. Reservation
- Table Name:       [TABLE_NAME]       ← e.g. reservations
- Route Slug:       [ROUTE_SLUG]       ← e.g. reservations
- Route Prefix:     [ROUTE_PREFIX]     ← e.g. front-desk.reservations
- Is New Module:    [YES|NO]           ← YES = new module, NO = add to existing
- Fields:           [FIELD_LIST]       ← e.g. title:string, room_id:foreignId, note:text|nullable
- Relationships:    [RELATIONSHIPS]    ← e.g. belongsTo Room
- Status Enum:      [STATUS_VALUES]    ← e.g. pending, in_progress, done
- Has Events/Jobs:  [YES|NO]

## Backend Specs

### Migration
- id, [FIELD_LIST], timestamps, softDeletes if needed
- foreignId → constrained()

### Enum
```php
<?php declare(strict_types=1);
namespace App\Enums;

enum [MODEL_NAME]Status: string {
    case Pending = 'pending';
    // match [STATUS_VALUES]
    public function label(): string { ... }
}
```

### Model
- `declare(strict_types=1)`
- `$fillable`, `$casts` (enum cast for status), relationships typed
- SoftDeletes if applicable

### Data/DTO
- Typed properties for all [FIELD_LIST]
- Match pattern in `app/Modules/FrontDesk/Data/ReservationData.php`

### Requests
- `authorize()` → `$this->user()->can(...)`
- `rules()` fully typed
- UpdateRequest uses `sometimes` where appropriate

### Resource
Always output:
```php
'id'           => $this->id,
'status'       => $this->status->value,
'status_label' => $this->status->label(),
'created_at'   => $this->created_at?->toISOString(),
'updated_at'   => $this->updated_at?->toISOString(),
// all [FIELD_LIST] fields
```

### Policy
- viewAny, view, create, update, delete, restore, forceDelete

### Observer
- created, updated, deleted hooks
- Dispatch events if Has Events = YES

### Action
- Single public `handle()` method
- Receives Data object, returns Model

### Service
- `paginate(array $filters)` — filterable by [FIELD_LIST] searchable fields
- `find(int $id): [MODEL_NAME]` — throws ModelNotFoundException
- `create([MODEL_NAME]Data $data): [MODEL_NAME]` — uses Action
- `update(int $id, [MODEL_NAME]Data $data): [MODEL_NAME]`
- `delete(int $id): void`
- DB::transaction() when writing multiple tables

### API Controller
- Thin — only: authorize, validate, call service, return response
- Class lives at `Api/V1/[MODEL_NAME]Controller.php` — no ApiController suffix
- Response format: `{ status: 1|0, data: ..., message: string }`
- Pagination response:
```json
{
  "status": 1,
  "data": {
    "items": [],
    "pagination": { "current_page": 1, "per_page": 15, "total": 0, "last_page": 1 }
  },
  "message": "..."
}
```
- HTTP codes: 200 / 201 / 204 / 422 / 404

### Web Controller
- Returns Inertia responses only
- Class lives at `Web/[MODEL_NAME]Controller.php` — no WebController suffix
- `index()`  → `Inertia::render('[FEATURE_NAME_PLURAL]/Index')`
- `create()` → `Inertia::render('[FEATURE_NAME_PLURAL]/Create')`
- `edit()`   → `Inertia::render('[FEATURE_NAME_PLURAL]/Edit',   ['item' => ...])`
- `show()`   → `Inertia::render('[FEATURE_NAME_PLURAL]/Show',   ['item' => ...])`

### Routes
```php
// api.php
// Namespace: App\Modules\[MODULE_NAME]\Http\Controllers\Api\V1\[MODEL_NAME]Controller
Route::middleware('auth:sanctum')
    ->prefix('v1/[ROUTE_SLUG]')
    ->name('[ROUTE_PREFIX].')
    ->group(fn() => [
        Route::get('/',        [[MODEL_NAME]Controller::class, 'index'])  ->name('index'),
        Route::post('/',       [[MODEL_NAME]Controller::class, 'store'])  ->name('store'),
        Route::get('/{id}',    [[MODEL_NAME]Controller::class, 'show'])   ->name('show'),
        Route::put('/{id}',    [[MODEL_NAME]Controller::class, 'update']) ->name('update'),
        Route::delete('/{id}', [[MODEL_NAME]Controller::class, 'destroy'])->name('destroy'),
    ]);

// web.php
// Namespace: App\Modules\[MODULE_NAME]\Http\Controllers\Web\[MODEL_NAME]Controller
Route::middleware('auth')
    ->prefix('[ROUTE_SLUG]')
    ->name('[ROUTE_PREFIX].')
    ->group(fn() => [
        Route::get('/',          [[MODEL_NAME]Controller::class, 'index']) ->name('index'),
        Route::get('/create',    [[MODEL_NAME]Controller::class, 'create'])->name('create'),
        Route::get('/{id}',      [[MODEL_NAME]Controller::class, 'show'])  ->name('show'),
        Route::get('/{id}/edit', [[MODEL_NAME]Controller::class, 'edit'])  ->name('edit'),
    ]);
```

## Frontend Specs

### Types
- Import `ApiResponse`, `PaginatedResponse` from `@/Types/api.ts`
- Define: `[MODEL_NAME]`, `[MODEL_NAME]Status`, `Create[MODEL_NAME]Dto`, `Update[MODEL_NAME]Dto`, `[MODEL_NAME]Filters`
- camelCase in all TS types

### Mappers
- `mapTo[MODEL_NAME]()` → API snake_case → UI camelCase
- `mapTo[MODEL_NAME]Payload()` → UI camelCase → API snake_case

### Pinia Store
- State shape:
```ts
items:        [MODEL_NAME][]
selectedItem: [MODEL_NAME] | null
loading:      boolean        // save/delete
loadingList:  boolean        // fetch
error:        string | null
pagination: {
    data: [MODEL_NAME][]
    meta: { current_page, last_page, per_page, total }
}
```
- Use mapper functions for API ↔ UI conversion
- Guard with `Array.isArray` before map/filter

### Composable
- `options?: { autoFetch?, initialFilters?, pollingInterval? }`
- Use `useLoading`, `useMessage`, `usePolling`
- `inject('$toast')`, `inject('$confirm')`
- Delete confirms via `$confirm.show({ variant: 'danger', ... })`
- Never expose store directly

### Pages — MANDATORY PATTERN
Before writing ANY .vue page:
1. MUST read `resources/js/Pages/Hotels/Create.vue` (highest priority — same module)
2. MUST read `resources/js/Pages/Reservations/Create.vue` (i18n pattern reference)
3. STRICTLY follow:

   **Permission (MANDATORY):**
   - `usePermission()` → `permission.check('create [MODEL_NAME_PLURAL]')`
   - `canCreate` computed → wrap entire form in `v-if="canCreate"`
   - `onMounted` → redirect to `/[ROUTE_SLUG]` if no permission

   **i18n Translations (MANDATORY):**
   - `import { useI18n } from '@/Composables/useI18n'`
   - `const { t } = useI18n()`
   - ALL user-facing strings MUST use `t('key')` — NO hardcoded strings
   - Labels, placeholders, button text, page title, hints — all via `t()`
   - `submitLabel` computed → `t('actions.creating')` / `t('actions.create')`

   **Form (MANDATORY):**
   - `useForm()` shape and field typing (with proper TS types e.g. `'' as number | string`)
   - `validateForm()` + `validateInertiaForm()` pattern
   - `form.clearErrors()` before submit
   - `form.setError()` for backend 422 errors
   - `mapBackendField()` for snake_case → camelCase error mapping
   - `scrollToFirstError()` after validation failure
   - `isSaving` computed from `form.processing || saving.value`
   - `Number(result.status) === 1` check
   - `form.reset()` before `router.visit()`
   - `watch()` for dependent field re-validation (if applicable)

4. DO NOT:
   - Use `reactive()` for form state
   - Use `reactive()` for errors
   - Use hardcoded strings — always `t()`
   - Skip `mapBackendField()`
   - Skip `scrollToFirstError()`
   - Skip permission check + `v-if="canCreate"`
   - Skip `form.reset()` before `router.visit()`

- **Index.vue**  : search, status filter, `<Table>`, pagination, edit/delete per row
- **Create.vue** : `useForm()` + `usePermission()` + `useI18n()` — follow Hotels/Create.vue pattern
- **Edit.vue**   : receives `item` as Inertia prop, pre-filled form, same pattern as Create.vue
- **Show.vue**   : read-only detail view, back + edit buttons
- All use `<AppLayout>`, `<AppButton>`, `<AppInput>`

### Page Pattern (Create/Edit)
```ts
// ✅ Standard pattern — MUST match resources/js/Pages/Hotels/Create.vue
import { computed, watch, onMounted } from 'vue'
import { useForm, router } from '@inertiajs/vue3'
import { usePermission } from '@/Plugins/directives/permission'
import { useI18n } from '@/Composables/useI18n'
import { required, minValue, validateInertiaForm } from '@/Utils/validation'
import { use[MODEL_NAME]s } from '@/Composables/[MODULE_NAME]/use[MODEL_NAME]s'

// ─── i18n (MANDATORY — no hardcoded strings) ─────────────
const { t } = useI18n()

// ─── Permission (MANDATORY) ──────────────────────────────
const permission = usePermission()
const canCreate  = computed(() => permission.check('create [MODEL_NAME_PLURAL]'))

// ─── Composable ──────────────────────────────────────────
const { create, saving } = use[MODEL_NAME]s()

// ─── Form (MANDATORY — useForm, NEVER reactive) ──────────
const form = useForm({
    field1: '' as number | string,
    field2: '',
    // ... all [FIELD_LIST] fields with proper TS types
})

// ─── Computed ────────────────────────────────────────────
const isSaving    = computed(() => form.processing || saving.value)
const submitLabel = computed(() =>
    isSaving.value ? t('actions.creating') : t('actions.create')
)

// ─── watch for dependent field re-validation (if needed) ─
watch(() => form.dependentField, () => {
    if (form.otherField) validateOtherField()
})

// ─── onMounted permission guard ──────────────────────────
onMounted(() => {
    if (!canCreate.value) router.visit('/[ROUTE_SLUG]')
})

// ─── Submit ──────────────────────────────────────────────
async function submit(): Promise<void> {
    form.clearErrors()
    if (!validateForm()) {
        scrollToFirstError()
        return
    }
    try {
        const result = await create({ ...form })
        if (Number(result.status) === 1) {
            form.reset()
            router.visit('/[ROUTE_SLUG]')
        }
    } catch (err: unknown) {
        const apiErr = err as Record<string, any>
        if (apiErr?.response?.data?.errors) {
            const backendErrors: Record<string, string[]> = apiErr.response.data.errors
            Object.entries(backendErrors).forEach(([key, messages]) => {
                form.setError(mapBackendField(key) as any, messages[0])
            })
            scrollToFirstError()
        }
    }
}

// ─── Validation ──────────────────────────────────────────
function validateForm(): boolean {
    return validateInertiaForm(form, {
        field1: [required],
        field2: [required, minValue(0)],
        // ... all required [FIELD_LIST] fields
    })
}

// ─── snake_case (Laravel) → camelCase (form) mapping ─────
function mapBackendField(field: string): string {
    const map: Record<string, string> = {
        field_one: 'fieldOne',
        field_two: 'fieldTwo',
        // ... all [FIELD_LIST] mappings
    }
    return map[field] ?? field
}

// ─── Scroll to first error ────────────────────────────────
function scrollToFirstError(): void {
    setTimeout(() => {
        const firstError = document.querySelector('.border-red-500')
        firstError?.scrollIntoView({ behavior: 'smooth', block: 'center' })
    }, 100)
}
```

**Template must-haves:**
```html
<!-- Permission guard -->
<div v-if="canCreate"> ... </div>

<!-- All labels via t() — NO hardcoded strings -->
<label>{{ t('hotels.field_label') }}</label>
<input :placeholder="t('hotels.field_placeholder')" />
<button :disabled="isSaving">{{ submitLabel }}</button>
```

**useForm benefits (why NOT reactive):**
- `form.errors` — built-in per-field error tracking
- `form.processing` — built-in loading state
- `form.reset()` — reset to initial values
- `form.clearErrors()` — clear all errors
- `form.setError()` — set errors from API response
- `form.isDirty` — track unsaved changes
